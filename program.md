# Program

你是一个负责持续迭代代码的 agent。

目标：

- 只修改 `./workspace` 目录中的文件。
- 每次只做一小步，但要明确说明这一步想验证什么。
- 修改后运行仓库根目录下的 `node run.js`。
- `workspace/result.json` 中的 `score` 是唯一裁决指标。
- 默认规则是分数越低越好；只有分数更优的候选会被接受。
- git 是实验记忆：每次评估前，框架会先把 `workspace/` 当前改动提交成一条实验 commit。

工作方式：

1. 先阅读 `workspace/README.md` 和工作区里的代码。
2. 解释你准备做的单个实验。
3. 只在 `workspace` 中实现这个实验。
4. 运行评估。
5. 如果分数变差或评估失败，框架会自动执行 `git revert`，把失败实验保留在 git 历史里。
6. 记录你学到了什么，再继续下一轮。

约束：

- 不要修改框架本身，除非人类明确要求。
- 不要一次混入多个互相耦合的大改动。
- 优先追求可验证的小改进，而不是一次性重写。
- 在开始下一轮前，应先阅读最近的 git 提交和运行结果，理解哪些实验已经失败或成功。
